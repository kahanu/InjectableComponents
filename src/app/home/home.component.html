This is an Angular 4.x demo application that demonstrates how to implement dynamic components in your workflow.

What are Dynamic Components?

Dynamic components is a concept. It's a way of instantiating a component (that already exists) and injecting it into another component on the fly.

Why is this important? We can already place a component directive in the HTML of another component.

This is true, and this is absolutely useful, but what if you need to change the referenced component based on the parent? You could use an ngSwitch statement, but that can lead to some maintenance.
