This is an Angular 4.x demo application that demonstrates how to implement dynamic components in your workflow.

What are Dynamic Components?

Dynamic components is a concept. It's a way of instantiating one or more components (that already exists) and injecting them into another component on the fly.

Why is this important?

We can already place a component directive in the HTML of another component. This is true, and this is absolutely useful, but what if you need to change the injected component with another component? You could use an ngSwitch statement, but that can lead to some maintenance. The better option is to dynamically inject components into a location in your view on the fly.

That's what this example will demonstrate. You'll get a better idea of how this works when you run the demos.

Demo Examples

There are two demo examples of how you can use Dynamic Components in Angular.

  1. Inline - the Companies area demonstrates how to be able to display the content from several components via a button bar, where the content is shown on the page in a specific area. This uses the ComponentFactoryResolver class to dynamically resolve the selected component and place it in a target HTML element.
  2. Dialog - you can also display dynamic components in a modal dialog as shown in the Suppliers area.
